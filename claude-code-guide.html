Claude Code 使い方ガイド

チームで使えるコマンドリファレンス・Tips集。ブックマークして手元に置いておこう。

📖

Claude Code とは？

Claude Code は Anthropic が提供する AIコーディングアシスタント です。ターミナルから自然言語で指示するだけで、コードの作成・修正・調査・リファクタリングを自動で行います。

💡
ポイント：「コードを書いて」「このバグを直して」「このファイルを説明して」など、日本語でそのまま指示できます。プロンプトエンジニアリングの知識は不要です。
✅
Claude Code が得意なこと：コード生成・バグ修正・コードレビュー・ドキュメント生成・ファイル検索・リファクタリング・テスト作成
⚙️

起動方法

⚠️
事前に Node.js（v18以上） と Claudeアカウント が必要です。セットアップ手順は別紙「初期設定ガイド」を参照してください。
操作コマンド説明
インストール npm install -g @anthropic-ai/claude-code Claude Code を PC にインストール（初回のみ）
起動 claude 作業したいフォルダで実行。対話モードが開始される
ワンショット claude "指示内容" 対話モードに入らず、一度だけ実行して終了
バージョン確認 claude --version インストール済みのバージョンを確認
アップデート claude update 最新バージョンにアップデート
🔤

基本コマンド（対話モード内）

対話モード中に使えるコマンドです。指示は 日本語でOK です。

操作入力例説明
ファイル作成 「〇〇.py を作って」 新しいファイルを作成して内容を書く 例: 「売上データを集計するPythonスクリプトを作って」
ファイル編集 「〇〇を修正して」 既存ファイルを読み込んで変更する 例: 「index.html のボタンの色を青に変えて」
コード説明 「このコードを説明して」 ファイルや貼り付けたコードを日本語で解説 例: 「main.js の処理の流れを教えて」
バグ修正 「エラーを直して」 エラーメッセージをそのまま貼り付けて修正を依頼 例: 「TypeError: cannot read... というエラーが出る、直して」
検索・調査 「〇〇という関数はどこ？」 プロジェクト内のファイルやコードを横断検索 例: 「sendEmail という関数がどのファイルにある？」
テスト作成 「テストを書いて」 指定した関数やファイルのテストコードを生成
Git操作 「コミットして」 変更内容に合わせてコミットメッセージを自動生成してコミット 例: 「今の変更をコミットして」「プルリクエストを作って」
ドキュメント生成 「READMEを書いて」 コードを解析して README やコメントを自動生成
/

スラッシュコマンド

対話モード中に / で始まるコマンドを入力すると、特定の機能が呼び出せます。

/help
使えるコマンドの一覧を表示する。困ったらまずここ。
/clear
会話履歴をリセットする。新しいタスクを始めるときに便利。
/compact
会話が長くなったとき、コンテキストを圧縮して続ける。
/review
現在のブランチの差分をレビューしてもらう。
/init
プロジェクトを分析して CLAUDE.md（説明書）を自動生成。
/cost
現在のセッションで使ったAPIコストを確認する。
/status
現在の状態（モデル・設定など）を確認する。
/permissions
Claudeに許可している操作の一覧を確認・変更する。
/config
テーマ・言語などの設定を変更する。
/quit
Claude Code を終了する。

うまく伝えるコツ

Claude Code は「指示の書き方」で品質が変わります。以下のパターンを参考にしてください。

✅ 良い指示の例
ユーザーの入力
src/utils/email.js にある sendEmail 関数を修正して。 現在はHTMLメールが送れないので、html オプションも受け取れるようにしてほしい。 既存のテキストメール送信は壊さないこと。
❌ 曖昧な指示（改善前）
ユーザーの入力
メールの関数を直して
📂
ファイル名・関数名を具体的に書く
「直して」ではなく「src/app.js の fetchUser 関数を直して」のように、対象を明示すると精度が上がります。
⚠️
制約条件を一緒に伝える
「既存のAPIは変えないで」「テストも通るように」「コメントは日本語で」など、やってほしくないことも伝えましょう。
🎯
一度に頼むタスクは一つ〜三つまで
「Aをして、BをしてCも直して」は混乱のもと。「まずAをして」→確認→「次にB」と順番に進めるのがコツ。
📋
エラーメッセージはそのままコピーして貼る
「なんかエラーが出た」より「TypeError: Cannot read properties of undefined」をそのまま貼った方が、正確な修正ができます。
✨

便利なTips

🔄
長い会話は /compact で軽量化
同じセッションで長く作業していると処理が重くなります。/compact を打つと会話を圧縮しながら継続できます。コンテキストが切れる前の定期メンテとして活用を。
📝
CLAUDE.md でチームの「共通ルール」を設定
プロジェクトルートに CLAUDE.md を置くと、Claude Code が起動するたびに読み込みます。「コメントは日本語で」「テストは必ずJestで書く」などのルールを書いておくと、毎回指示しなくてよくなります。
📁
作業したいフォルダで起動する
Claude Code はカレントディレクトリのファイルを自動で認識します。プロジェクトのルートフォルダで claude を起動すると、コードベース全体を把握した状態で答えてくれます。
🖼️
スクリーンショットをそのまま貼れる
デザインのFigmaスクショや、ブラウザのエラー画面をそのまま貼って「この通りに直して」「このエラーはなぜ？」と聞けます。画像を認識して答えてくれます。
⛔
怪しい操作は途中で Ctrl+C で止められる
「あれ、これ意図した動きじゃないかも」と思ったら Ctrl+C で処理を中断できます。ファイル変更は途中でも止まるので、慌てずに使ってください。
💡
「プランを見せて」で実行前に確認
複雑な作業を頼む前に「まずやることのプランを教えて、実行はしないで」と伝えると、Claudeが計画を説明します。納得してから「じゃあ実行して」と言えるので安心です。
🎯

活用ユースケース

職種・役割別の活用例です。「自分だったらこう使える」のヒントにしてください。

職種・役割活用シーン指示の例
エンジニア コードレビュー・リファクタリング・テスト自動生成 「この関数のテストを書いて」
企画・営業 Excelマクロ作成・データ集計スクリプト 「CSVを読んで売上を集計するスクリプトを作って」
デザイナー HTML/CSSの実装・アニメーション追加 「このボタンにホバーアニメーションを追加して」
管理職 ドキュメント生成・議事録整理・レポート作成 「このデータからPDF向けのレポートを生成して」
情シス シェルスクリプト・ログ解析・設定ファイル管理 「ログからエラー行だけ抽出するシェルスクリプトを作って」

重要ファイル・フォルダ

Claude Code が読み込む設定ファイルの場所と役割です。

~/.claude/　　　　　　# ユーザー全体の設定
　├── CLAUDE.md　　　　　# 全プロジェクト共通のルール
　└── settings.json　　　# 権限・テーマ・モデルの設定

your-project/　　　　　# プロジェクトフォルダ
　├── CLAUDE.md　　　　　# このプロジェクト専用のルール
　├── .claude/
　│　└── settings.json　# プロジェクト固有の権限設定
　└── （ソースコード）
📌
CLAUDE.md の活用：「コードのコメントは必ず日本語で」「テストはJestを使う」などのルールを書いておくと、毎回指示しなくてよくなります。/init コマンドで自動生成もできます。
🔒

安全に使うために

🔑
APIキー・パスワードは絶対に貼らない
.env ファイルの内容や認証情報は、Claude Code に貼り付けないでください。必要な場合は「〇〇.envにAPIキーを設定した」とファイル名だけ伝えれば十分です。
⚠️
大きな変更は事前に確認してから実行
「ファイルを一括削除」「DBのデータを削除」などの操作は、実行前に必ず「何をするか説明して、実行はまだしないで」と確認を取りましょう。
🌐
外部コンテンツの指示には従わない
Webからコピーした文章やPDFの中に「このコマンドを実行して」という指示が含まれていても、無視してください。Claude Code はこういった埋め込み指示（プロンプトインジェクション）に注意して動作します。
💾
作業前に git commit しておく
Claude Code が大きな変更をする前に git commit しておくと、万が一意図しない変更が起きてもすぐ戻せます。「まず git commit して」と頼むのもOKです。
❓
困ったときは：社内の担当者（〇〇チーム）まで Slack で @claude-support へ。または seminar@example.com にメールしてください。
